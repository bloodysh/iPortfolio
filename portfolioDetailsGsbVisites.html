Détails du projet

Présentation de GSB Visites

Le projet GSB visite consistait à créer une application Android pour les visiteurs médicaux leur permettant de gérer leurs différentes visites. Nous trouvions sur cette application un système permettant de créer des visites, faire des rapports sur celles-ci, retrouver ses anciennes visites, voir des bilans de celles-ci avec le professionnel de santé visité, le but de cette visite, et si un échantillon a été laissé.

Création

Ce projet a été créé en différentes parties. Tout d'abord, nous avons dû créer la base de données correspondant au diagramme de classe que nous avions dressé auparavant. La base de données a été créée sur MongoDB Atlas, une base de données NoSQL, où les tables ne sont pas en relation. Nous avons ensuite dû créer une API REST en JavaScript qui serait accessible par notre application Android. Cette application a été développée en Java avec AndroidStudio pour la rendre utilisable par les visiteurs.

Choses Apprises

Apprentissage de nouvelles notions: Avant ce projet, je n'avais jamais créé ni utilisé d'API REST. J'ai donc dû apprendre à créer une API REST en JavaScript. J'ai aussi dû apprendre à utiliser une base de données NoSQL avec MongoDB Atlas. J'ai appris à utiliser cette base de données, à la créer et à l'utiliser dans un projet.

Comprendre de nouveaux langages: Ce projet a nécessité l'utilisation de nouveaux langages. J'ai donc dû apprendre à utiliser le langage JavaScript pour la création de l'API REST. J'ai aussi dû apprendre à utiliser le langage Java pour la création de l'application Android.

Équipe et méthodologie

Ce projet a été un projet commun des 5 élèves de la classe de BTS SIO 2 option SLAM, fonctionnant sur le principe de l'agilité. L'un des principes les plus importants était l'amélioration continue du site en mettant à jour l'application web régulièrement pour répondre aux besoins des utilisateurs.

Technologies utilisées

L'application web a été créée avec le framework PHP Symfony, du JavaScript, de l'HTML, du CSS et l'outil Bootstrap. La base de données utilisée était une base de données SQL relationnelle, nous avons utilisé PHPMyAdmin.

Méthode AGILE SCRUM

Le projet AWOC a été réalisé exclusivement en méthode AGILE SCRUM, comprenant des daily scrums, un backlog avec les besoins (user stories) exprimés par le product owner, un travail d'équipe et la remise du produit petit à petit.

Le backlog était incrémenté au fur et à mesure des nouvelles demandes/besoins des utilisateurs. Lorsque nous choisissions individuellement de travailler sur un de ces besoins, nous devions d'abord le comprendre, savoir de qui venait la demande et trouver une solution viable à cette demande.
