Détails de projet

Besoin spécifique de GSB

GSB est une entreprise fictive, qui nous a adressé un besoin spécifique. Le but de ce projet était de créer une application web permettant à des visiteurs médicaux de déclarer leurs frais forfaitaire (déplacement, restauration, nuitée, étapes) et hors forfait via des notes de frais à renseigner depuis l'application web. L'application web devait permettre à un comptable de valider ou non les frais déclarés par les visiteurs médicaux. Le projet a été réalisé en méthode AGILE SCRUM avec notamment l'utilisation d'un backlog et de users stories.

Déroulement du projet

Ce projet a été l'un des projets principaux de notre année. Nous sommes parties d'une base, l'ancienne application de GSB - frais. Nous avions de plus un cahier des charges à partir duquel nous avons créé le backlog nécessaire au projet.

Technologies utilisées

L'application web a été créée en majorité en PHP, HTML et CSS avec notamment le framework Symfony, ce qui a aussi impliqué l'utilisation de Twig pour les vues. Le projet repose sur le principe de Modèle Vue Contrôleur (MVC) pour faciliter la compréhension du code par un développeur tiers.
